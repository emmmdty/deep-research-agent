# Phase 23 — Native benchmark console

## Objective
Expose native benchmark artifacts and selected run actions in the GUI.

## Required outcomes
- smoke_local / regression_local overview page
- suite cards for company12 / industry12 / trusted8 / file8 / recovery6
- scorecard view
- casebook view
- links to manifests / summaries / results markdown
- optional trigger buttons for bounded local native runs if safe

## Acceptance
- a reviewer can understand native benchmark layers from the GUI alone
- native scorecard and casebook are visible in-app
- if trigger actions are implemented, they are clearly labeled local and bounded

## Validation
- GUI benchmark page smoke
- at least one casebook/report deep link works
