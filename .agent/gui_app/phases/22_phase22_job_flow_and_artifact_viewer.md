# Phase 22 — Research job UX

## Objective
Implement end-to-end local research-job UX in the GUI.

## Required outcomes
- new research form
- source-profile selector
- jobs list
- job detail page
- progress/event panel
- report viewer
- artifact/bundle inspector

## Acceptance
- a user can submit a job from the GUI
- a user can inspect status and artifacts
- at least one real local job path is documented end to end
- screenshots or docs for the main flow exist

## Validation
Run at least:
- one local API-backed GUI smoke flow
- one end-to-end manual or scripted bounded check
