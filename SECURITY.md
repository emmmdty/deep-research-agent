# Security Policy

## Supported Versions

This project is maintained on the `main` branch. Security fixes, when available, are applied there first.

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for security problems.

Instead:

1. Use GitHub private vulnerability reporting if it is enabled for the repository.
2. If private reporting is unavailable, contact the maintainer through GitHub and include:
   - a short description of the issue
   - reproduction steps
   - impact assessment
   - any suggested mitigation

You should receive an acknowledgement after review. Public disclosure should wait until the issue has been assessed and, when possible, fixed.
